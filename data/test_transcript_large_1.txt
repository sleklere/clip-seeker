 C++, un idioma de programación compilado estéticamente, famoso por su uso amplio en la infraestructura de software y famoso por su curva de aprendizaje profunda. Fue creado en 1979 por Bjorn Straustrup en AT&T Bell Labs. Fue inspirado por la naturaleza de la simulación, pero necesitaba un idioma con una alta performance de C, y así, C con clases, fue nacido. Se diseñó como un super set de C, lo que significa que casi cualquier programa de C es también un programa de C++, pero también añade abstracciones sin sobrevaluación, como patrones orientados a objetos, como polimorfismo, encapsulación y heredidad. Hoy en día se usa en una gran variedad de sistemas con demandas de memoria constrictas, como el engine Unreal para juegos de videojuegos AAA, programas como Adobe After Effects que fueron usados para editar este video, databases como MySQL y MongoDB, sistemas embedidos como el display en tu toaster de computadoras, y es usado para implementar mucho de nuestros juegos de videojuegos de nivel bajo. También se usa para implementar infraestructura, como compiladores de lenguaje y máquinas virtuales. Es tan popular porque ofrece control de memoria y hardware de nivel bajo como C, pero con abstracciones de nivel alto como clases y pointers inteligentes. C++ hace que sea más difícil dispararte en el pie, pero cuando lo haces, te va a matar. Para empezar, instala un compilador de C++ como GCC o Clang, luego crea un file terminando en CPP. Para imprimir Hello World, simplemente usamos standard, y el output de character followed by the bitwise shift left operator and a string literal. Podemos eliminar este STD, por supuesto, añadiendo un namespace a la parte superior del file. Ahora, si queremos crear una variable de string, podemos hacerlo al escribirla como un array de personajes. Sin embargo, si incluimos string de la biblioteca, podemos usar el type string. El cambio real, por supuesto, es el hecho de que C++ apoya programación orientada a objetos con clases. Una clase es solo un blueprint para un objeto. Dentro de una clase, puedes definir atributos y métodos que serán privados por default, o hacerlos públicos. También es posible definir métodos fuera de la definición de la clase usando un doble colon. Los métodos también pueden ser definidos múltiples veces con diferentes parámetros, lo que se conoce como overloading y es una forma de polimorfismo. Además, podemos definir constructores y destructores para ejecutar código cuando un objeto es creado o destruido. Las clases también apoyan la heredidad, permitiendo a los desarrolladores compartir la lógica más eficientemente a través de un programa. Para instanciar un objeto, escoge el nombre de la clase, seguido por el nombre de tu objeto, y opcionalmente cualquier parámetro para el constructor. Cuando se trata de memoria, podríamos manejar todo manualmente con parámetros y referencias. Sin embargo, herramientas como UniquePointer pueden hacer el trabajo de una manera más fácil y segura, asegurándonos de que solo un objeto puede ser adquirido a memoria.
